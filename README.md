## 1. 프로젝트 개요

본 프로젝트는 한국디자인진흥원(KIDP) 공식 홈페이지를 분석하고,  
기존 사이트의 정보 구조와 사용자 경험(UI/UX)을 개선하는 방향으로  
메인 페이지와 주요 서브 페이지를 리뉴얼한 개인 포트폴리오 프로젝트입니다.

공공기관 사이트 특성상 정보 전달이 핵심이라는 점에 집중하여  
콘텐츠 우선순위, 탐색 흐름, 반응형 환경을 고려한 구조로 재설계하였습니다.

- 작업 기간: 약 5일  
- 작업 형태: 개인 프로젝트 (기획 / 디자인 / 퍼블리싱 / 배포 전부 담당)

---

## 2. 문제 인식 (Why)

기존 홈페이지를 분석하며 아래와 같은 문제점을 발견했습니다.

- 사이트에 처음 진입했을 때 **기관이 무엇을 하는 곳인지 한눈에 파악하기 어려움**
- 공지, 사업, 소식, 입찰 정보가 혼재되어 있어 **정보 탐색 동선이 복잡함**
- 콘텐츠 양에 비해 정보 우선순위가 명확하지 않아 **스크롤 피로도가 높음**
- 모바일 환경에서의 사용성을 고려한 **반응형 구조가 충분하지 않음**
- 공공기관 사이트임에도 불구하고 **메인 페이지에서 핵심 정보로 빠르게 접근하기 어려움**

이 프로젝트는 단순한 디자인 변경이 아니라,  
“정보를 어떻게 보여줘야 사용자가 덜 고민하는가”에 대한 문제 해결을 목표로 시작되었습니다.

---

## 3. 프로젝트 목표 (Goal)

본 리뉴얼 프로젝트의 목표는 다음과 같습니다.

1. 메인 페이지에서 기관의 역할과 핵심 정보를 빠르게 이해할 수 있도록 구조 개선  
2. 공지 / 사업 / 소식 등 성격이 다른 정보를 명확히 구분한 정보 구조 설계  
3. 데스크톱과 모바일 환경 모두에서 동일한 정보 접근 경험 제공  

---

## 4. 정보 구조(IA) 및 기획 의도

### 메인 페이지 기획 방향
- 첫 화면에서 **기관 성격 + 주요 소식**을 동시에 인지할 수 있도록 구성
- 자주 찾는 정보(공지, 입찰, 설문 등)는 하단이 아닌 **메인 흐름 내에 배치**
- 시각적 요소는 강조 수단으로 사용하고, 정보 자체가 묻히지 않도록 설계

### 내비게이션 구조
- 공공기관 사이트에서 익숙한 상위 카테고리 구조 유지
- 메뉴 수를 줄이기보다는 **카테고리 내 정보 묶음의 논리성**에 집중
- 아이콘 기반 바로가기 영역을 통해 주요 사업 정보에 빠르게 접근 가능하도록 구성

---

## 5. 주요 구현 내용

- 메인 페이지 중심의 콘텐츠 허브 구조 구현
- 슬라이드/배너를 활용하여 정보 밀도 조절
- 섹션 단위로 명확한 시각적 구분을 적용해 스크롤 흐름 개선
- 모바일 환경을 고려한 반응형 레이아웃 적용
- 공지, 입찰, 설문 등 성격이 다른 콘텐츠를 카드형 리스트로 분리 표현

---

## 6. 기술 스택

- HTML / CSS / JavaScript  
- Swiper.js (콘텐츠 슬라이드 구현)  
- GitHub Pages (정적 배포)

외부 빌드 도구 없이 결과물을 빠르게 확인할 수 있도록  
정적 사이트 환경을 기준으로 구현하였습니다.

---

## 7. 작업 범위 및 역할

본 프로젝트는 100% 개인 작업으로 진행되었습니다.

- 사이트 분석 및 기획
- 정보 구조(IA) 설계
- UI 구성 및 퍼블리싱
- 반응형 레이아웃 구현
- GitHub Pages 배포

---

## 8. 결과 및 배운 점

이번 프로젝트를 통해 단순히 화면을 구현하는 것보다  
**정보를 어떤 순서와 구조로 보여주는지가 사용자 경험에 더 큰 영향을 준다**는 점을 체감했습니다.

특히 공공기관 사이트의 경우,
- 정보 우선순위 설정
- 콘텐츠 분류 기준
- 사용자 관점의 탐색 흐름

이 세 가지가 매우 중요하다는 것을 실제로 고민하며 설계할 수 있었습니다.

---

## 9. 한계 및 개선 방향

- 실제 사용자 테스트를 통한 검증은 진행하지 못함
- 웹 접근성(WA) 기준을 충분히 반영하지 못함
- CMS 연동 없이 정적 페이지로만 구성됨

향후에는  
웹 접근성 기준을 반영하고, 실제 사용자 시나리오 기반 테스트를 통해  
정보 구조를 더 정교하게 개선해보고자 합니다.
